# 共有フォルダ整理プログラム運用マニュアル

- 文書版: Ver.260607
- 作成日: 2026-06-07
- 対象環境: Windows / Z ドライブ運用

## 1. 目的
本マニュアルは、共有フォルダ整理のデイリー運用で実行される処理範囲、主な挙動、確認方法、運用ルールを定義する。

## 2. デイリー運用で実行されるタスク
現在のデイリー運用は、以下 2 タスクを実行する。

1. GDX_DailyRun
2. OTHER_DailyRun

### 2.1 GDX_DailyRun
- 起動: run_gdx_wrapper.bat -> run_gdx_logged.ps1 -> run_gdx.py -> 91GDX・252WORKNO-program/cli.py
- 主処理:
  - Drive 取り込み（設定時）
  - GDExtraction から 91 への投入
  - 工番マスタに基づくリネーム
  - 91 配下の B1/B2/B3/B4 整理
  - 画像連番化、画像圧縮、不要ファイル整理
- 主な設定値:
  - 画像圧縮上限 max_kb: 1000KB（91GDX 側）

### 2.2 OTHER_DailyRun
- 起動: run_91other_wrapper.bat -> run_91other_logged.ps1 -> run_91other.py -> 91OTHER-program/cleanup.py -> 91OTHER-program/91フォルダ以外整理
- 主処理:
  - 91 以外の一般メディアフォルダ整理
  - 連番リネーム（001_YYMMDD 形式）
  - 画像圧縮
  - 英単語アンダースコア補正（英単語間を半角スペースへ）
- 主な設定値:
  - 画像圧縮上限 max_kb: 2048KB（91OTHER 側）

## 3. 処理対象範囲（重要）

### 3.1 Z 全体にかかるか
- 現在設定では Z 全体にはかからない。
- デイリー処理の実質対象は 2to9_業務別フォルダ 配下。

### 3.2 GDX 側の対象
- 2to9_業務別フォルダ配下の以下ルートが対象。
  - 91_工番別実績写真・動画
  - 25_リビルト・中古機/252_整備資料
  - 92_PO LIST
  - 97_技術資料/978_CADデータ図庫/9781_工事工番

### 3.3 OTHER 側の対象
- base_dir は Z:\takachiho。
- ただし allowed_top_dirs が ["2to9_業務別フォルダ"] のため、実走査は 2to9_業務別フォルダ 配下に限定。
- 備考: allowed_top_dirs を空にすると base_dir 全体走査になる実装。

## 4. ファイル名正規化ルール

### 4.1 Windows 禁止文字
- 禁止文字はアンダースコアへ置換する。
- 対象文字: \\ / : * ? " < > |

### 4.2 全角/半角の扱い
- NFKC 正規化は、主に 91 整理のフェーズ判定（B1/B2/B3/B4 分類）で使用。
- 全ファイル名を一律で全角->半角変換する仕様ではない。

### 4.3 英単語アンダースコア補正
- before_after のような英単語間アンダースコアは before after に補正。
- ただし連番や日付フォーマット（例: 001_240101）は維持。

## 5. 画像・動画処理

### 5.1 連番リネーム
- 形式: 001_YYMMDD.ext（または工番プレフィックス付き）
- 日付は EXIF/動画メタデータ/mtime の順で判定。

### 5.2 画像圧縮
- JPG/JPEG はサイズ閾値以下ならスキップ。
- PNG/HEIC/HEIF は必要に応じて JPG へ変換。
- 圧縮後の目標上限は設定値 max_kb。

### 5.3 連番・圧縮の適用単位
- GDX 側: 91 配下の対象フォルダ整理時に適用。
- OTHER 側: allowed_top_dirs で許可された走査ルート配下で適用。

## 6. B4 関連運用（91 整理）
- B4 の空フォルダ削除を実施。
- B4 配下サブディレクトリの空判定・削除を実施。
- junk ファイル（Thumbs.db など）除去を実施。
- B1/B2/B3 内の非メディアを B4 へ退避する処理あり。

## 7. 同期運用ポリシー
- Y ドライブへの手動同期は現在不要（運用方針）。
- 基本はスクリプト更新を git 管理する。
- 不要同期フォルダは都度手動削除で対応。

## 8. 日次確認手順
1. タスク状態確認
   - GDX_DailyRun
   - OTHER_DailyRun
2. 実行ログ確認
   - logs/gdx_run_*.txt
   - logs/other_run_*.txt
3. 注意ログ確認
   - attention 系レポート
4. 失敗時は権限エラー、ファイルロック、ネットワークドライブ状態を優先確認。

### 8.1 タスク状態確認コマンド
```powershell
schtasks /Query /TN "GDX_DailyRun" /V /FO LIST
schtasks /Query /TN "OTHER_DailyRun" /V /FO LIST
```

## 9. 代表的な障害と対処

### 9.1 Access denied / Permission denied
- 原因: ファイルロック、権限不足。
- 対処: 関連アプリ終了、管理者権限で再実行、再起動後再実行。

### 9.2 File not found（削除中）
- 原因: 既に別処理で削除済み。
- 対処: 多くは無害。最終的な残存有無を再確認する。

### 9.3 HEIC 非対応
- 原因: pillow-heif 未導入など。
- 対処: 実行環境の依存関係を確認し、必要パッケージを導入。
