## 改定に反映すべき変更点

### 1. デイリーラン統合(実行経路の全面書き換え)
run_gdx_logged.ps1 に統合: GDX整理 → OTHER整理 → AzCopy Blob同期(RAG用+271指令書PDF) → lw_blob_sync → ld_sort → アノテーション同期 → RAGインデックス(run_rag_index.py 自動実行) → タスクステータスBlob記録

### 2. ログのY:共有運用
- 実行ログをY:\...\logs へ30秒ごとリアルタイム同期(Start-Job+UNCパス、FileShare.ReadWrite、UTF-8明示)
- 対象: dailyrun_*.txt、photo_video_91_*.log(ルート直下含む)
- 日次確認は「ノートからY:のログを見る」手順に変更

### 3. 2PC運用・スケジュール実行の技術要件
- .runtime のロックファイルで二重実行防止、`--force` でスキップ可
- GDX_RUNTIME_ROOT = C:\ProgramData(SYSTEMユーザー実行対応)
- タスク登録はUNCパス推奨、bat は %~dp0 化・UTF-8

### 4. RAG検索アプリ(TSEG FM SEARCH)※新規章
- 写真・図面のAI検索(Streamlit、Azure App Service)
- 工事マスタCSV連携: 納入先・請求先の絞り込み、工番正規化
- 271_修理工事指令書: PDF検索+リネーム(工番_工事名_指令書)
- 工番検索は workno eq フィルタ(誤マッチ防止)、フレーズクエリ
- manifest.json 空スキャン時の上書き防止(Z:未接続対策)

### 5. LINE WORKS Bot群 ※新規章
- 学習協力Bot(アノテーション): 対象フェーズ B2・B3のみ(B1・B4除外)、Tトリガー、Y/N/X操作、24h放置自動スキップ、品質チェック
- 定時配信: 朝の挨拶08:05、夕方リマインダー16:55、写真投稿ランキング、休日スキップ(lw_holiday.json)
- ld_sort.py: LWExtraction → <工番>/B4 振り分け、空フォルダ削除
- アノテーションJSONは _annotations/<工番>/ に一元管理
- ユーザー名: Directory APIから取得、Blobに24hキャッシュ

### 6. 確認タスクの追加

- タスクステータスをBlobに記録、check_task_status.py で確認
- 06:00 daily_runs_summary/attention(既存)

### 7. 障害対処の追記候補

## 運用メモ(260711夕方の知見)

- ノートからのログ監視は `Y:\管理本部\情報管理課\tseg_vscode\Zフォルダ整理\logs\` を直接参照する(Claudeサンドボックス経由は約15分キャッシュ遅延あり)
- `--force` の連打で複数ランが並走し得る(Pythonプロセス2個並走を確認)。並走中はrename失敗(WinError 2)等のレースが起きるため、前ランの完全終了を確認してから再実行する
- トランスクリプト(dailyrun_*.txt)は Python 実行中に出力がフラッシュされず、リアルタイム進捗は photo_video_91_*.log で見る
- トランスクリプト内のPython出力の二重文字化け(UTF-16系)は未解決(PS1自身の出力は正常)

- KEIRI-PCに ffprobe(ffmpeg)未導入 → 動画の日時取得がmtime代用になっている(リネーム日付精度に影響、導入検討)
- 停止で残った __TMP__ ファイルは次回ランが「stale TMP回収」で自動復旧する(手動対応不要)
- GitHub Copilotクラウドエージェントにはローカル実行・Y:参照系のタスクを渡さない(サンドボックスでアクセス不可、クレジット浪費)
